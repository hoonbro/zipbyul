# 집별 — 청약 안전마진 기획안 v1

## 청약 공고에 "당첨되면 깔고 가는 시세차익"을 자동 계산해 보여주는 차별화 기능 — 기획안 (검토본)

- 문서 버전: **v1 (검토본 · 데이터 타당성 확인 기준)**
- 작성 기준: 2026.06
- 짝 문서: `집별_서비스기획안_v2.1_최종.md`, `집별_백엔드기획안_v2.1_최종.md`
- 범위: **MVP는 Tier 1(안전마진 추정 1지표)로 한정.** Tier 2·3은 로드맵으로만 기술.

------

## 0. 배경 · 목표

집별은 현재 "놓치지 않게 해주는 알림형 리서치 앱"이다(서비스기획안 1-3). 청약 공고를 **모아서 알려주긴 하지만, 그 공고가 좋은 물건인지는 사용자가 직접 판단**해야 한다. 경쟁 앱(청약홈·호갱노노)도 "공고 나열"까지만 한다.

차별화 가설:

> **청약 공고에 "예상 안전마진"을 한 줄로 붙이면, 단순 알림 앱에서 "판단을 돕는 앱"으로 한 단계 올라간다.**

안전마진은 청약 커뮤니티의 사실상 표준 의사결정 지표다. 이 지표를 **자동·투명하게** 계산해 공고 카드에 배지로 노출하는 것이 본 기획의 목표다.

------

## 1. 핵심 지표 정의

### 1-1. 안전마진 공식 (MVP 확정)

```
예상 안전마진 = 인근 시세(전용면적 매칭, 최근 6개월 실거래 중앙값) − 실분양가(주택형별 공급금액)
```

- **시세**: 보유 중인 국토부 실거래가(`real_estate_transactions`)를 사용. 외부 시세 API 미사용(2장 참조).
- **분양가**: 청약홈 주택형별 공급금액(2장). MVP는 **기본형 분양가**만 사용하고 옵션·확장비는 제외(고지로 명시).
- **매칭 단위**: MVP는 **같은 법정동(`dong_name`) + 전용면적 ±5㎡** 구간. 반경 매칭은 2차.

### 1-2. 표시 등급 (배지)

| 등급 | 기준(인근 시세 대비) | 배지 |
| ---- | ---- | ---- |
| 높음 | 분양가가 시세보다 **15%↑ 저렴** | 🟢 안전마진 높음 |
| 보통 | 0~15% 저렴 | 🟡 안전마진 보통 |
| 낮음/없음 | 시세와 같거나 비쌈 | ⚪ 마진 낮음 |
| 산출 불가 | 인근 실거래 표본 부족(< N건) | "비교 데이터 부족" |

> **표본 부족 시 억지로 숫자를 만들지 않는다.** 표본이 적은 신축 택지지구·재개발 단지가 많으므로, `산출 불가` 상태를 1급 시민으로 둔다.

### 1-3. 함께 노출하는 맥락 (오해 방지)

안전마진 숫자만 보면 오해하므로 아래를 같이 표기한다.

- 비교에 쓴 **기준 단지·표본 건수·기간**(계산 근거 노출)
- **규제로 묶이는 기간**: 전매제한·실거주의무 여부(헤더 `분양가상한제` 플래그 + 규제 룰표). "마진이 있어도 N년 못 판다"를 같이 보여줘야 의미가 산다.

## 2. 데이터 소스 — 현황 vs 추가

### 2-1. 결론: **신규 API 키 발급 불필요**

안전마진 MVP에 필요한 모든 데이터는 **이미 보유한 키·소스로 확보 가능**하다. 빠진 것은 "새 API"가 아니라 **이미 쓰는 청약홈 서비스의 다른 엔드포인트**다.

| 필요 데이터 | 소스 | 현재 상태 |
| ---- | ---- | ---- |
| 청약 공고 헤더·일정 | 청약홈 `getAPTLttotPblancDetail` | ✅ 사용 중 |
| **주택형별 분양가** | 청약홈 **`getAPTLttotPblancMdl`** (같은 서비스 15098547) | ❌ **미사용 — 추가 호출 필요** |
| 인근 시세 | 국토부 실거래가(MOLIT) | ✅ `real_estate_transactions` |
| 규제 플래그(상한제) | 공고 헤더 `분양가상한제` 필드 | ✅ 수집되나 미저장 |
| 대출이자 시뮬(2차) | 한국은행 ECOS 금리 | ✅ 보유 |

### 2-2. 추가 호출 1건 — `getAPTLttotPblancMdl`

- 무순위는 `getRemndrLttotPblancMdl` (대칭 구조).
- **기존 `DATA_GO_KR_SERVICE_KEY` 그대로 사용** — 신청·심사 불필요.
- ⚠️ **착수 전 확인 1건**: 응답의 공급금액 필드명(`SUPLY_AMOUNT` 계열로 추정)을 라이브 1회 호출로 확정해야 한다. fixture(헤더)에는 금액이 없어 미확정.

### 2-3. 향후 선택 (MVP 제외)

| 기능 | 추가 필요 | 비고 |
| ---- | ---- | ---- |
| 가점 컷 예측 | 청약홈 경쟁률 서비스(별도 등록) | Tier 3 |
| 반경 매칭 정밀화 | 지오코딩(카카오/도로명주소) | Tier 2, 동 단위로 시작하면 불필요 |
| 청약↔실거래 단지 연결 | 국토부 공동주택 단지코드 | 텍스트 매칭 오류가 실문제일 때만 |

> **제외 확정**: KB·네이버부동산 단지 시세는 공개 API가 없어 스크래핑이 필요 → 약관·법적 리스크로 **채택 안 함**. 시세는 실거래가로 충분.

## 3. 신규 스키마 (초안)

분양가는 "주택형(타입)별 1:N"이므로 헤더 테이블에 못 넣는다. 별도 테이블 신설.

```sql
-- V8 (초안) — 공고별 주택형 단위 (분양가 정규화)
CREATE TABLE announcement_unit (
  id              bigserial PRIMARY KEY,
  announcement_id bigint NOT NULL REFERENCES housing_announcements,
  house_type      varchar(40),      -- 주택형 (예: 084.9842A)
  area_m2         numeric(7,2),     -- 전용면적
  supply_count    int,              -- 공급세대수
  supply_amount_manwon bigint,      -- 분양가(기본형, 만원)
  dedup_hash      varchar(64) NOT NULL,
  collected_at    timestamptz NOT NULL DEFAULT now(),
  UNIQUE (dedup_hash)
);
CREATE INDEX idx_unit_ann ON announcement_unit(announcement_id);
```

- `housing_announcements`에 `price_cap_yn boolean`(분양가상한제) 컬럼 추가 — 이미 헤더로 들어오는 값.
- 안전마진은 **저장하지 않고 조회 시 계산**(시세는 매일 변하므로). 부하가 문제되면 일배치로 `unit`별 스냅샷 캐시.

------

## 4. 산출 로직 (서버)

```
공고 상세 조회 시, 주택형(unit)마다:
  1. unit.area_m2 ± 5㎡, 같은 dong_name, 최근 6개월 SALE 실거래 조회
  2. 표본 < N건 → '산출 불가' 반환
  3. 시세 = 표본 가격 중앙값
  4. 마진 = 시세 − unit.supply_amount, 비율 = 마진 / 시세
  5. 1-2 등급표로 배지 매핑
  6. 응답에 계산근거(기준동·표본수·기간) 동봉
```

- 위치: Spring 서빙 레이어(JdbcClient + raw SQL, 영속화 전략 메모 준수).
- N(최소 표본) 초기값 **5건** — 운영하며 튜닝.

## 5. 화면 · UX (목업 연동 필요)

- **공고 카드(리스트)**: 대표 1줄 배지 — `🟢 안전마진 +1.2억 (인근 시세 대비)`. 대표값은 **세대수 최다 주택형** 기준.
- **공고 상세**: 주택형별 표 — `주택형 | 전용 | 분양가 | 인근시세 | 마진 | 등급`.
- **고지 문구(필수)**: "추정치이며 투자 권유가 아닙니다. 옵션·확장비 제외, 실거래 표본 기반. 투자 판단의 책임은 사용자에게 있습니다."
- 상세 화면에 **계산 근거 펼치기**(기준 단지·표본·기간) — 투명성이 신뢰의 핵심.

## 6. 단계별 범위

| 단계 | 내용 | 비고 |
| ---- | ---- | ---- |
| **MVP (Tier 1)** | `Mdl` 분양가 수집 → `announcement_unit` → 안전마진 1지표 + 배지 + 고지 | 신규 API 없음 |
| Tier 2 | 실투자금 타임라인(계약금/중도금/잔금 + ECOS 금리 이자), 규제 묶임기간 상세, 입주시점 예상시세(실거래 추세) | 신규 API 없음 |
| Tier 3 | 가점 컷 예측, 세금 시뮬, 반경 매칭 | 추가 API/룰 필요 |

## 7. 리스크 · 법적 고지

1. **투자 권유 해석 리스크** — "안전마진/예상시세"는 투자자문업으로 오인될 수 있다. 전 지표에 **추정치·참고용 고지** 필수, 숫자는 보수적으로, 근거 투명 공개.
2. **시세 표본 편향** — 신축·택지지구는 인근 실거래가 거의 없다. `산출 불가`를 정상 상태로 처리(억지 추정 금지).
3. **분양가 정의 모호** — 옵션·확장·발코니가 빠진 기본형 분양가임을 명시. 실제 납부액과 다를 수 있음.
4. **필드명 미확정** — `Mdl` 응답 구조는 착수 전 라이브 검증 필요.
