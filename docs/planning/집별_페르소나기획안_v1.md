# 집별 — 페르소나 기획안 v1

## 5개 핵심 페르소나의 니즈를 상세 분석하고, 각자가 집별을 통해 "얻고자 하는 것"을 정리한 문서

- 문서 버전: **v1 (검토본 · 구현 현황 정합화 기준)**
- 작성 기준: 2026.06
- 짝 문서: `집별_서비스기획안_v2.1_최종.md`, `집별_백엔드기획안_v2.1_최종.md`, `집별_안전마진기획안_v1.md`
- 범위: 서비스기획안 §3-2의 5개 사용자 유형을 페르소나로 구체화하고, 정보 니즈 → 의사결정 → 현재 제공 현황을 연결한다.
- 원칙: 집별은 **매물 앱이 아니라 알림형 리서치 앱**이며, 제외 타깃(단기 투자·갭투자·트레이딩)은 본 문서에서도 다루지 않는다.

------

## 0. 분석 프레임

각 페르소나를 5개 축으로 분석한다.

1. **정체성** — 누구인가 (상황·자금·시급성)
2. **핵심 동기** — 집별을 켜는 진짜 이유 (Job-to-be-done)
3. **정보 니즈** — 의사결정에 필요한 정보 목록
4. **얻고자 하는 것** — 정보를 넘어 최종적으로 원하는 결과(outcome)
5. **제공 현황** — ✅ 제공 중 / 🟡 보유 데이터로 가능(미구현·진행 중) / 🔴 추가 데이터 필요

> 제공 현황은 현재 코드(`web/src/lib/types.ts`, `Home.tsx`, `margin/`, 캘린더 배지)·수집기(`adapters/`)·기획안을 교차 확인한 기준이다.

## 2. P1 · 청약·분양 대기자

### 2-1. 정체성

서울 무주택 30~40대. 청약통장 보유, 가점이 애매하거나(40점대) 추첨제·무순위를 병행 노린다. 청약홈·호갱노노·오픈채팅방을 매일 들락거리며 "줍줍(무순위)" 정보를 놓칠까 불안해한다. **공고 1건을 놓치면 다음 기회까지 수개월**이라 시급성이 가장 높은 페르소나.

### 2-2. 핵심 동기 (JTBD)

> "관심 단지의 접수일을 놓치지 않고, 이 공고가 **'당첨되면 남는 물건'인지**를 빠르게 판단하고 싶다."

단순 알림을 넘어 "이 공고가 좋은가?"의 판단까지 원한다. 안전마진 기능의 1차 타깃.

### 2-3. 정보 니즈

- 관심지역 신규 민간분양·무순위·오피스텔 공고
- 접수 시작 / 서류제출 / 당첨자 발표 / 계약 일정 (단계별 D-Day)
- 공고 핵심: 공급세대수, 주택형별 분양가, 자격(가점·거주요건), 원문 링크
- **당첨 시 예상 시세차익(안전마진)** + 계산 근거(표본·기간·기준단지)
- **규제 묶임 기간**: 분상제·전매제한·실거주의무 ("마진 있어도 N년 못 판다")
- (희망) 경쟁률·가점 컷 예측

### 2-4. 얻고자 하는 것

| 정보 니즈 | 최종 outcome |
| ---- | ---- |
| 일정 알림 | **놓침 없는 청약 캘린더** — 불안 해소 |
| 안전마진 배지 | **"넣을 가치가 있나"의 즉답** — 판단 비용 절감 |
| 규제 기간 | **"넣어도 묶이는 기간" 인지** — 잘못된 청약 방지 |
| 경쟁률(희망) | **당첨 가능성 가늠** — 전략적 청약 |

### 2-5. 제공 현황

| 정보 | 현황 | 비고 |
| ---- | ---- | ---- |
| 신규 공고 알림·단계별 D-Day | ✅ | 청약홈 수집 + 캘린더 + `urgentEvents`(dDay/stars) + 푸시 |
| 공고 핵심 구조화 정보 | ✅ | `AnnouncementDetail.summary` |
| **안전마진 배지·주택형별 표** | 🟡 진행 중 | `margin/SafetyMarginService`, V8 마이그레이션, `announcement_unit.py` **미커밋 작업 중**. 분양가 엔드포인트(`getAPTLttotPblancMdl`) 라이브 호출로 필드 확정 필요 |
| 분상제·무순위 단서 칩 | ✅ | `CalendarItem.priceCap`/`unranked` 배선 완료 |
| **전매제한·실거주 N년 룰표** | 🟡 | 헤더값 → 규제 룰 매핑 미구현. 안전마진의 신뢰성을 좌우 |
| 경쟁률·가점 컷 예측 | 🔴 | 청약홈 경쟁률 서비스 별도 등록 필요 (Tier 3, 의도적 제외) |

## 3. P2 · 공공임대 대기자

### 3-1. 정체성

소득·자산 요건을 충족하는 무주택자. 청년·신혼·고령 등 특별공급 자격을 갖췄거나 갖추려 한다. 행복주택·국민임대·매입임대·전세임대가 주 관심. 자격 조건이 까다롭고 공고가 비정기라 **"내가 되는 공고만" 거르는 것**이 핵심 고통.

### 3-2. 핵심 동기 (JTBD)

> "내 소득·자산·세대 조건으로 신청 가능한 공공임대 공고가 언제 어디서 뜨는지, 마감 전에 알고 싶다."

### 3-3. 정보 니즈

- LH 행복주택·국민임대·매입임대·전세임대 신규 공고/마감
- 소득·자산·무주택·거주 요건 (자격 핵심 필드)
- 단지 위치·입주 시점·임대료 수준
- (희망) SH 청년안심주택·장기전세

### 3-4. 얻고자 하는 것

| 정보 니즈 | 최종 outcome |
| ---- | ---- |
| LH 공고 알림 | **자격 맞는 임대 기회 포착** |
| 자격 요건 정리 | **"내가 되나" 빠른 자가진단** |
| SH 공고(희망) | **공공임대 커버리지 완성** |

### 3-5. 제공 현황

| 정보 | 현황 | 비고 |
| ---- | ---- | ---- |
| LH 공고/마감 알림 | ✅ | `adapters/lh.py` + 캘린더 |
| 자격 요건(소득·자산·무주택) | ✅ (구조화 한도) | `summary` 자격 필드. 공고 원문 구조 의존 |
| 단지 위치·입주·임대료 상세 | 🟡 | 마이홈포털 단지정보 **보조 검토 단계**, MVP 미연동 |
| **SH 청년안심주택·장기전세** | 🔴 | 공식 API 불명확 → 2차. 온보딩 선택은 되나 "개발 예정" 빈 상태 |

## 4. P3 · 실거주 매수 검토자

### 4-1. 정체성

자금이 어느 정도 있고 "내 집 마련" 실수요. 청약과 일반 매수를 저울질하며 **"지금 사야 하나, 더 기다려야 하나"**의 타이밍 고민이 핵심. 금리·심리·실거래 흐름에 민감하지만 분석 앱을 깊게 파지는 않는다.

### 4-2. 핵심 동기 (JTBD)

> "금리 방향, 시장 심리, 내 관심지역 실거래 흐름을 한곳에서 보고 매수 타이밍을 가늠하고 싶다."

### 4-3. 정보 니즈

- 한국은행 기준금리 결정 일정·결과
- 서울 집값전망 심리(CSI) 현재값·추이
- 관심지역 실거래 신규 등록(매매가 흐름)
- (희망) 대출이자·실투자금 시뮬, 입주시점 예상시세, 시장온도

### 4-4. 얻고자 하는 것

| 정보 니즈 | 최종 outcome |
| ---- | ---- |
| 금리 일정 | **"금리 변곡점" 인지** — 자금 계획 |
| 집값전망 심리 | **시장 분위기 체감** — 매수 타이밍 보조 |
| 관심지역 실거래 | **내 동네 시세 감각** — 가격 판단 |
| 시뮬(희망) | **실부담액 가늠** — 의사결정 정량화 |

### 4-5. 제공 현황

| 정보 | 현황 | 비고 |
| ---- | ---- | ---- |
| 기준금리 결정 일정 | ✅ | seed 등록 + 캘린더/정책 카드 |
| 서울 집값전망 심리(CSI) | ✅ | `marketOutlook` 카드 + `HousePriceOutlook` 추이 |
| 관심지역 실거래 신규 등록 | ✅ | `recentTransactions`(매매/전월세·면적·금액·층) |
| **대출이자/실투자금 시뮬** | 🟡 | ECOS 금리 **보유**, 미구현 (안전마진 Tier 2) |
| **입주시점 예상시세** | 🟡 | 실거래 추세로 산출 가능, 미구현 (Tier 2) |
| **시장온도·전세온도** | 🔴 | `SoonCard` "개발 예정". 국토연구원 지수 라이선스 필요 (2차) |

## 5. P4 · 전월세 이사 준비자

### 5-1. 정체성

전월세 계약 만료가 다가와 곧 이사해야 한다. 매수보다 **전월세 시세 + 공공임대 기회**가 관심. 동(洞) 단위로 좁은 지역을 본다. 임대 기회가 있으면 갈아탈 의향도 있다.

### 5-2. 핵심 동기 (JTBD)

> "내 동네 전월세가 요즘 얼마에 나가는지 감을 잡고, 갈아탈 만한 공공임대 기회를 놓치지 않고 싶다."

### 5-3. 정보 니즈

- 관심지역(가능하면 법정동) 전월세 신규 실거래 등록
- 공공임대(전세임대 등) 신청 기회
- (희망) 전월세 시세 추세 비교

### 5-4. 얻고자 하는 것

| 정보 니즈 | 최종 outcome |
| ---- | ---- |
| 전월세 등록 | **내 동네 임대 시세 감각** — 계약 협상력 |
| 공공임대 기회 | **저렴한 임대 갈아타기** |
| 동 단위 좁히기 | **노이즈 없는 내 동네만** |

### 5-5. 제공 현황

| 정보 | 현황 | 비고 |
| ---- | ---- | ---- |
| 관심지역 전월세 신규 등록 | ✅ | `recentTransactions` tradeType 구분 |
| 공공임대(전세임대) 기회 | ✅ | LH 전세임대 수집 |
| **법정동 단위 좁히기** | 🟡 | `watchRegions.bjdCode` 스키마 존재. **실거래는 동단위 가능, 공고는 헤더에 법정동 미적재(주소→자치구만)** → 공고 동매칭은 수집 보강(주소→법정동 파싱) 필요 |
| 전월세 시세 비교/추세 | 🔴 | 단건 등록만 제공. 분석은 MVP 의도적 제외 |

## 6. P5 · 시장 모니터링러

### 6-1. 정체성

당장 거래 계획은 없지만 부동산 흐름은 챙기고 싶은 라이트 유저. 앱을 길게 안 본다. **"매일 아침 하루 요약 1건"**이 hero 경험인 핵심 타깃이자, 집별의 재방문율을 떠받치는 페르소나.

### 6-2. 핵심 동기 (JTBD)

> "직접 검색하지 않아도, 오늘 서울 부동산에서 중요한 것만 30초 안에 받고 싶다."

### 6-3. 정보 니즈

- 오늘 스캔 요약 (한 줄 hero)
- 중요도 별점 (걸러보기)
- 하루 요약 1건 (푸시)
- 관심지역 한눈 요약
- 출처·갱신시각·신고지연 고지 (신뢰)
- (희망) 부동산 뉴스·AI 요약

### 6-4. 얻고자 하는 것

| 정보 니즈 | 최종 outcome |
| ---- | ---- |
| 하루 요약 | **검색 없이 흐름 파악** — 최소 노력 |
| 별점 필터 | **중요한 것만** — 피로 없는 정보 |
| 출처·고지 | **믿을 수 있는 정보** — 신뢰 |

### 6-5. 제공 현황

| 정보 | 현황 | 비고 |
| ---- | ---- | ---- |
| 오늘 스캔 요약(hero) | ✅ | Home 레이더 "오늘 N건 스캔" |
| 중요도 별점 | ✅ | `StarRating`, `baseScore`/`stars` |
| 하루 요약 1건 | ✅ | `dailyDigest` enum + 알림 디스패처 |
| 관심지역 한눈 요약 | ✅ | `regionSummary`(공고/마감/실거래 카운트) |
| 출처·갱신시각·신고지연 고지 | ✅ | `dataFreshness.notices` |
| **부동산 뉴스·AI 요약** | 🔴 | News 페이지 전체 "개발 예정"(2차). 저작권·품질 부담 |

| P3 실거주 매수 | 금리 일정·집값전망·실거래 | 이자/투자금 시뮬, 예상시세 | 시장온도·전세온도 |
| P4 전월세 | 전월세 등록·전세임대 | **공고 법정동 매칭(수집 보강)** | 전월세 추세 분석 |
| P5 모니터링러 | 하루요약·별점·요약·고지 | — | 뉴스·AI 요약 |

### 7-1. 핵심 인사이트

1. **5개 페르소나 공통 코어(캘린더·D-Day·집값전망·실거래·하루요약·고지)는 모두 작동한다.** MVP 골격은 완성 단계.
2. **가장 임팩트 큰 갭 2개는 신규 API 키 없이 보유 소스로 닫을 수 있다.**
   - **P1 안전마진** — 코드 진행 중, 분양가 엔드포인트 라이브 검증만 남음. 집별의 차별화 한 줄(`알림 → 판단`)을 완성하는 기능.
   - **P4 공고 법정동 매칭** — 수집단 주소→법정동 파싱 보강. 안전마진 동 단위 매칭(P1)과도 직결.
3. **🔴 항목 대부분은 기획안에서 2차/Tier 3로 의도적 제외**한 것(SH·시장온도·경쟁률·뉴스)이며, 라이선스·약관·별도 등록 리스크가 본질. MVP 범위 일탈이 아니다.

### 7-2. 우선순위 제언

| 순위 | 과제 | 수혜 페르소나 | 비용 | 근거 |
| - | ---- | ---- | ---- | ---- |
| 1 | 안전마진 마감(분양가 라이브 검증 + 룰표) | P1 | 낮음(진행 중) | 차별화 핵심, 신규 API 불필요 |
| 2 | 공고 주소→법정동 파싱 수집 보강 | P1·P4 | 중 | 동 단위 매칭·노이즈 감소의 전제 |
| 3 | 이자/실투자금 시뮬 (ECOS 활용) | P3 | 중 | 보유 데이터, Tier 2 |
| 4 | SH·시장온도·뉴스 | P2·P3·P5 | 높음 | 2차 — 라이선스/약관 선결 |
