## 1. 라우팅 맵

`OnboardingGate`(설정 조회 후 게이팅) → 그 아래 온보딩(네비 없음)과 `Layout`(하단 탭) 분기.

| path | 화면 | 데이터 | X-Anonymous-Id |
| ---- | ---- | ---- | ---- |
| `/onboarding` | 온보딩 | `PUT /v1/preferences` | O |
| `/` (index) | 홈(오늘의 레이더) | `GET /v1/feed/home` | O |
| `/news` | 뉴스(2차 · "개발 예정" 안내) | — | - |
| `/market-outlook` | 지수(집값 전망 심리, lazy) | `GET /v1/market-index/house-price-outlook` | - |
| `/calendar` | 통합 캘린더 | `GET /v1/calendar?from&to` | - |
| `/announcements` | 공고 목록 | `GET /v1/announcements?supplyType&page&size` | - |
| `/announcements/:id` | 공고 상세 | `GET /v1/announcements/{id}` | - |
| `/watch` | 관심지역 | `GET /v1/watch/regions/summary` + prefs | O |
| `/settings` | 알림 설정 | prefs · `GET /v1/notifications` · devices · me/data | O |

- 루트 라우트 `errorElement = <RouteError />` (렌더/로더 오류 방어).
- 하단 탭(5종, 목업 정합): **레이더(홈) · 캘린더 · 뉴스 · 지수 · 설정**.
- `/announcements`·`/watch`는 탭에서 제외 — **캘린더 공고 항목 딥링크**(`refType=ANNOUNCEMENT`)와 **홈 "내 관심지역 › 편집"**으로 진입.

## 2. 상태 관리

### 서버 상태 — TanStack Query

| queryKey | 훅 |
| ---- | ---- |
| `['feed','home']` | useFeedHome |
| `['preferences']` | usePreferences |
| `['market-index','house-price-outlook']` | useHousePriceOutlook |
| `['calendar', params]` | useCalendar |
| `['announcements', params]` | useAnnouncements |
| `['announcement', id]` | useAnnouncement |
| `['watch','summary']` | useWatchSummary |
| `['notifications']` | useNotifications |

뮤테이션: `useSavePreferences`(PUT) → `['preferences']` 세팅 + `['feed']`·`['watch']` 무효화. `useDeleteMyData`(DELETE).

### 로컬 / 클라이언트 상태

- `localStorage.jb_anonymous_id` — 익명 식별자(AppBoot가 최초 발급). 삭제 시 복구 불가.
- `localStorage.jb_device_id` — FCM 기기 등록 id(끄기/해제용).
- 폼 상태는 `useState` **lazy 초기화**(쿼리 데이터를 props로 받아 초기값) — `useEffect` 동기화 금지(eslint `react-hooks/set-state-in-effect`).

## 3. 데이터 레이어 (`src/lib`)

- **`api.ts`** — `apiFetch<T>(path, {method, body, withAnonymousId})`. `BASE_URL = VITE_API_BASE_URL ?? localhost:8080`(prod는 `''`=상대). 공통 에러 envelope→`ApiError{status,code,traceId}`, 204→undefined, `X-Anonymous-Id` 주입.
- **`anonymousUser.ts`** — `ensureAnonymousId`(없으면 `POST /v1/anonymous-users`), `getAnonymousId`, `clearAnonymousId`.
- **`firebase.ts`** — `isPushConfigured`, `registerPush`(권한→토큰→`POST /v1/devices`, SW 명시등록), `unregisterPush`, `onForegroundMessage`. config 없으면 graceful no-op.
- **`platform.ts`** — `isIos`, `isStandalone`.
- **`constants.ts`** — 서울 25구, 관심유형 9종(2차 2종 `available:false`), 알림강도 5종, event/supply 라벨맵, 공고 공급유형 필터 목록. enum 코드는 `contracts/enums.yaml` SSOT. (캘린더 그룹 분류는 `Calendar.tsx` `itemGroup`.)
- **`colors.ts`** — 카테고리 색(`categoryColor`)·D-day 색(`ddayLook`) 등 목업 표시 규칙.
- **`types.ts`** — 응답 타입(현재 수기 — `openapi.yaml` 코드젠으로 대체 검토).

## 4. 화면별 스펙

각 화면 공통: **로딩/에러/빈** 상태 명시.

1. **온보딩** — **4-step 위저드**(STEP n/4 · 진행바): ①관심지역 칩 → ②상황(2차·개발 예정 비활성) → ③관심유형 칩(9종, 2차 2종 비활성) → ④알림강도 카드. 스텝별 필수(지역≥1·유형≥1)로 진행, 마지막에 `PUT` → 홈. *("건너뛰기"는 온보딩 게이트 루프 방지로 제거.)*
2. **홈(레이더)** — 레이더 히어로(애니, 스캔 카운트=긴급+실거래/중요=별4↑/마감임박=D-3↓) · 시장요약 3카드(집값전망 live + 시장온도·전세온도 개발예정) · 중요일정(태그·D-day·별점) · 관심지역 요약 · 실거래 NEW · 신선도 고지.
3. **지수(집값전망)** — name/제공 중 · band 슬라이더(80~120, 중립 100) + 큰 값 + 전월비 + 추이 라인차트(Recharts, 다크) + 시장온도·전세온도 "개발 예정" 카드 + 출처/면책.
4. **캘린더** — 월간/D-Day 토글 · 그룹 필터(청약·공공임대·정책·금리, **클라 분류**) · 월간 그리드(점·범례·월 이동) + 리스트. 조회는 월 전체(월간)/+60일(D-Day), 공고 이벤트는 상세 딥링크.
5. **뉴스(2차)** — "개발 예정" 안내 화면(목업) + 2차 제공 예정 목록.
6. **공고 목록/상세** — 공급유형 필터 + 페이지네이션(이전/다음). 상세: 구조화 필드 + summary_json + 원문 링크, 404 처리. *(목업의 AI요약/일정 타임라인/별점은 백엔드 미제공으로 미구현.)*
7. **관심지역** — 요약(집계) + 자치구 편집(저장 시 prefs 병합 PUT).
8. **알림 설정** — 푸시 켜기/끄기(권한·iOS 안내) · 강도/유형/실거래옵트인/하루요약(시각)/방해금지 편집 · 알림함 · **내 데이터 삭제**.

## 5. 온보딩 게이팅
